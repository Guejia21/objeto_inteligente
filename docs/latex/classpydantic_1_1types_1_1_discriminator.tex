! abstract "{}\+Usage Documentation"{} \href{../concepts/unions.md\#discriminated-unions-with-callable-discriminator}{\texttt{Discriminated Unions with {\ttfamily Callable} {\ttfamily Discriminator}}}

Provides a way to use a custom callable as the way to extract the value of a union discriminator.

This allows you to get validation behavior like you\textquotesingle{}d get from {\ttfamily Field(\doxylink{classpydantic_1_1types_1_1_discriminator_a0a17d85717117b02387810a115f3aac7}{discriminator}=\texorpdfstring{$<$}{<}field\+\_\+name\texorpdfstring{$>$}{>})}, but without needing to have a single shared field across all the union choices. This also makes it possible to handle unions of models and primitive types with discriminated-\/union-\/style validation errors. Finally, this allows you to use a custom callable as the way to identify which member of a union a value belongs to, while still seeing all the performance benefits of a discriminated union.

Consider this example, which is much more performant with the use of {\ttfamily \doxylink{classpydantic_1_1types_1_1_discriminator}{Discriminator}} and thus a {\ttfamily Tagged\+Union} than it would be as a normal {\ttfamily Union}.
