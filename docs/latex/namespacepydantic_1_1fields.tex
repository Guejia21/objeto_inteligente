! abstract "{}\+Usage Documentation"{} \href{../concepts/fields.md\#the-computed_field-decorator}{\texttt{The {\ttfamily computed\+\_\+field} decorator}}

Decorator to include {\ttfamily property} and {\ttfamily cached\+\_\+property} when serializing models or dataclasses.

This is useful for fields that are computed from other fields, or for fields that are expensive to compute and should be cached.

Create a field for objects that can be configured.

Used to provide extra information about a field, either for the model schema or complex validation. Some arguments apply only to number fields ({\ttfamily int}, {\ttfamily float}, {\ttfamily Decimal}) and some apply only to {\ttfamily str}.

Note\+:
\begin{DoxyItemize}
\item Any {\ttfamily \doxylink{namespacepydantic_1_1fields_ab23dd2e726f156660828255ccccd78ee}{\+\_\+\+Unset}} objects will be replaced by the corresponding value defined in the {\ttfamily \doxylink{namespacepydantic_1_1fields_aa4486d527ccac10cd6f34c7562c1f67d}{\+\_\+\+Default\+Values}} dictionary. If a key for the {\ttfamily \doxylink{namespacepydantic_1_1fields_ab23dd2e726f156660828255ccccd78ee}{\+\_\+\+Unset}} object is not found in the {\ttfamily \doxylink{namespacepydantic_1_1fields_aa4486d527ccac10cd6f34c7562c1f67d}{\+\_\+\+Default\+Values}} dictionary, it will default to {\ttfamily None}
\end{DoxyItemize}

Args\+: default\+: Default value if the field is not set. default\+\_\+factory\+: A callable to generate the default value.
