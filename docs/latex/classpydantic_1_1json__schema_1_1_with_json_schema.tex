! abstract "{}\+Usage Documentation"{} \href{../concepts/json_schema.md\#withjsonschema-annotation}{\texttt{{\ttfamily With\+Json\+Schema} Annotation}}

Add this as an annotation on a field to override the (base) JSON schema that would be generated for that field. This provides a way to set a JSON schema for types that would otherwise raise errors when producing a JSON schema, such as Callable, or types that have an is-\/instance core schema, without needing to go so far as creating a custom subclass of \doxylink{classpydantic_1_1json__schema_1_1_generate_json_schema}{pydantic.\+json\+\_\+schema.\+Generate\+Json\+Schema}. Note that any {\itshape modifications} to the schema that would normally be made (such as setting the title for model fields) will still be performed.

If {\ttfamily \doxylink{classpydantic_1_1json__schema_1_1_with_json_schema_a0aadda42cb8f104507845d1d5adc0a13}{mode}} is set this will only apply to that schema generation mode, allowing you to set different json schemas for validation and serialization.
