! abstract "{}\+Usage Documentation"{} \href{../concepts/models.md\#dynamic-model-creation}{\texttt{Dynamic Model Creation}}

Dynamically creates and returns a new Pydantic model, in other words, {\ttfamily \doxylink{namespacepydantic_1_1main_ac8cdd2a08cb57f1cec43beb92713f321}{create\+\_\+model}} dynamically creates a subclass of \mbox{[}{\ttfamily \doxylink{classpydantic_1_1main_1_1_base_model}{Base\+Model}}\mbox{]}\mbox{[}\doxylink{classpydantic_1_1main_1_1_base_model}{pydantic.\+Base\+Model}\mbox{]}.

Args\+: model\+\_\+name\+: The name of the newly created model. {\bfseries{config}}\+: The configuration of the new model. {\bfseries{doc}}\+: The docstring of the new model. {\bfseries{base}}\+: The base class or classes for the new model. {\bfseries{module}}\+: The name of the module that the model belongs to; if {\ttfamily None}, the value is taken from {\ttfamily sys.\+\_\+getframe(1)} {\bfseries{validators}}\+: A dictionary of methods that validate fields. The keys are the names of the validation methods to be added to the model, and the values are the validation methods themselves. You can read more about functional validators \href{https://docs.pydantic.dev/2.9/concepts/validators/\#field-validators}{\texttt{here}}. {\bfseries{cls\+\_\+kwargs}}\+: A dictionary of keyword arguments for class creation, such as {\ttfamily metaclass}. {\bfseries{qualname}}\+: The qualified name of the newly created model. \texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}field\+\_\+definitions\+: Field definitions of the new model. Either\+: \begin{DoxyVerb}- a single element, representing the type annotation of the field.
- a two-tuple, the first element being the type and the second element the assigned value
  (either a default or the [`Field()`][pydantic.Field] function).
\end{DoxyVerb}


Returns\+: The new \mbox{[}model\mbox{]}\mbox{[}\doxylink{classpydantic_1_1main_1_1_base_model}{pydantic.\+Base\+Model}\mbox{]}.
